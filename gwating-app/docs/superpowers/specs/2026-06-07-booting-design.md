## 2. 비주얼 언어

### 방향: Soft Warm (화이트 베이스)
- **배경**: 순백색 `#ffffff`. 핑크 그라디언트는 히어로 영역에만 극히 제한적으로 사용
- **Primary**: `#ff5a6f` (코랄 핑크) — 버튼·로고·강조 텍스트에만 집중
- **Gradient**: `linear-gradient(135deg, #ff5a6f, #ff7e5f)` — CTA 버튼 전용
- **Shadow**: `0 4px 18px rgba(255,90,111,0.28)` — 그라디언트 버튼에만 적용
- **Font**: Pretendard Variable (기존 유지)
- **Border radius**: 버튼 `14px`, 카드 `16px`, 작은 요소 `10~12px`, pill `999px`
- **참고 앱**: 위피(Wipee), 아만다 — 친근하고 부드러운 한국 앱 톤

### 컬러 토큰 변경 없음
기존 `tailwind.config.ts`의 토큰 유지. 추가 토큰만 append:
```
"gender-m-bg": "#f0f5ff", "gender-m-text": "#4f9eff", "gender-m-border": "#cce0ff"
"gender-f-bg": "#fff0f2", "gender-f-text": "#ff5a6f", "gender-f-border": "#ffd6dd"
```

## 3. 페이지별 설계

### 3-1. 홈 (`/`)
**방식**: Ultra-minimal splash — 정보 나열 없음, 기능은 진입 후 자연스럽게 노출

**레이아웃**:
- 상단: 로고 (`⚡ 부팅`) + "부산대 과팅 서비스" 캡션
- 중앙: 원형 일러스트 링 (🎉, 나중에 로고 일러스트로 교체)
- 하단:
  - 태그라인: "당신의 연애세포를 **부팅**하세요!"
  - 서브카피: "부산대생끼리 팀을 이뤄 딱 맞는 상대팀과 설레는 과팅을"
  - CTA 버튼 (그라디언트): "성향 테스트 시작하기 →"
  - 텍스트 링크: "팀 초대를 받으셨나요? **코드로 합류**"

**제거**: 기존 "어떻게 진행되나요?" 스텝 섹션, 무드 칩 나열 → 모두 삭제

### 3-2. 성향 테스트 (`/test`)
**플로우**: 성별 선택 → 퀴즈 10문항 → 결과 + 닉네임 입력

#### 성별 선택 화면 (신규)
- 퀴즈 시작 전 단독 화면
- 카드 2개: 남자(🙋‍♂️) / 여자(🙋‍♀️) — 탭해서 선택
- 선택된 카드 테두리 컬러: 남자 `#4f9eff`, 여자 `#ff5a6f`
- "매칭 외 목적으로 사용되지 않아요" 안내 문구
- CTA: "다음 — 성향 테스트 시작 →"

#### 퀴즈 화면
- 상단: 그라디언트 프로그레스바 + "질문 N / 10" + 퍼센트
- 질문 텍스트: `Q. ` 빨간 접두사 + 본문
- 선택지: 탭 즉시 핑크 하이라이트 → 자동으로 다음 문항 (별도 확인 버튼 없음)

#### 결과 화면
- 역할 이모지 (🔥🎯🤍✨) 크게 표시
- 역할 이름 + 설명
- 성향 점수 바 차트 4개 (분위기 조율 / 배려심 / 적극성 / 소통 균형) — **신규**
- 닉네임 입력 → "팀 만들러 가기 →"

**저장**: `localStorage`에 `gender` 필드 추가 (`"male" | "female"`)

### 3-3. 팀 만들기 (`/team/create`)
**핵심 변경**: 역할 직접 지정 방식 → 초대 코드 방식으로 전면 교체

**레이아웃** (상→하 순서):
1. 팀 이름 입력 (placeholder: "예: 컴공 왕자들, 경영 여신들")
2. 나이대 입력
3. 원하는 과팅 분위기 선택 — 리더만 결정. 라디오 방식 (1개 선택)
4. 팀장(나) 카드 — 성향 테스트 결과 자동 표시 (닉네임 + 역할 + **성별 뱃지** 신규)
5. 팀원 초대 섹션:
   - 초대 코드 박스: `BT-XXXX` 형식 코드 + 복사 버튼 — **skeleton** (UI만, 기능 없음)
   - 합류 완료 팀원 카드: 닉네임 + 성별 뱃지 + 역할 칩 (테스트 완료 시 자동)
   - 대기 중 카드: "⏳ 초대 수락 대기"
   - "+ 팀원 더 초대하기" 버튼
6. 제출 버튼: 팀원 최소 1명 합류 완료 전 비활성

**제거**: 기존 `MemberRoleCard` (닉네임 + 역할 직접 입력) 컴포넌트 사용 중단

### 3-6. 날짜·시간 조율 (신규: `/match/schedule`)
**날짜 선택**:
- 달력 UI (현재 월 기준 +1달)
- 내가 선택한 날: 진한 핑크 (filled)
- 상대팀도 가능한 날: 연한 핑크 border (실시간 반영 — skeleton에서는 mock)
- 다음 버튼 → 시간 선택으로

**시간 선택**:
- 같은 방식으로 시간대 선택 (오후 5시 ~ 11시 버튼 그리드)
- 두 팀 겹치는 가장 빠른 시간 자동 확정

**매칭 로직** (백엔드):
- 두 팀장이 각각 가능 날짜 제출 → intersection 중 가장 빠른 날 자동 확정
- 시간도 동일 방식

### 3-8. Q&A 일일 공개 (신규: `/match/qa`)

#### 공개 분배 로직
```
하루 공개량 = ceil(총질문수 / 남은일수)
나머지는 초반 날에 배분
예) 5문항, 3일 → 2 / 2 / 1
```

#### 팀원 순번제
- 팀원을 순서대로 순번 배정 (팀장 → 팀원 순)
- **페어링**: 양 팀의 동일 순번 인덱스끼리 매핑 (A팀 1번 ↔ B팀 1번, A팀 2번 ↔ B팀 2번)
- 오늘 지정된 팀원에게 푸시 알림 (푸시는 skeleton)
- 상단 순번 타임라인: 완료(✓) / 오늘(⭐) / 대기(🔒) 시각화

#### 내 차례일 때
- 어두운 카드(다크 그라디언트)에 오늘의 질문 표시
- 선택지 4개 → 선택 후 "답변 제출하고 상대 보기 →" 버튼
- **공개 조건**: 양쪽 팀원이 모두 제출해야 서로의 답변 공개 (썸원 방식)
  - 내가 제출했지만 상대가 아직 → "상대팀 답변 대기 중..." 표시
  - 양쪽 모두 제출 완료 → 즉시 공개

#### 내 차례 아닐 때
- 오늘 담당 팀원 양쪽 모두 제출 완료된 경우에만 대화 구경 가능
- 아직 대기 중이면 "수진님이 아직 답변 중이에요 ⏳" 표시
- "내 차례가 되면 알림 드릴게요 🔔" 안내

#### 답변 공개 화면
- 내 답 vs 상대 답 나란히 비교
- **자동 코멘트**: 답변 패턴 기반 한 줄 코멘트 (예: "둘 다 먼저 말 거는 타입!")
- 다음 질문 예고 (잠금 카드)

#### 분위기별 질문 세트 (각 5문항 이상)
| 분위기 | 질문 예시 |
|--------|-----------|
| 편한 대화형 | "대화가 끊겼을 때 당신의 행동은?" / "절대 꺼내지 않을 주제는?" |
| 게임/술자리형 | "첫 게임으로 뭐 하고 싶어요?" / "술 마실 때 당신의 포지션은?" |
| 활발한 친목형 | "기억에 남는 사람의 특징은?" / "과팅 당일 본인의 역할은?" |
| 예의/안전 중시형 | "첫 만남에서 가장 불편한 상황은?" / "과팅 후 연락할 때 먼저 하는 편인가요?" |
| 자연스러운 소개팅형 | "소개팅과 과팅의 차이를 한 마디로?" / "첫 인상에서 가장 중요하게 보는 건?" |

#### D-1 전체 기록 화면
- 모든 팀원 쌍의 Q&A 히스토리 카드 리스트
- **케미 하이라이트**: 답변이 유사한 쌍에 자동 코멘트 (예: "지호↔태양 케미 주목!")
- "내일 만나요! 🎉" CTA

type UserProfile = {
  nickname: string;

### TeamMember에 gender 추가
```ts
type TeamMember = {
  nickname: string;
  role: MemberRole;
  traits?: Record<TraitKey, number>;
  isLeader?: boolean;
  gender: "male" | "female";  // 신규
};
```

### 매칭 조건에 성별 밸런스 추가
```ts
// 두 팀 합산 남:여 = 1:1 여야 매칭 가능
function isGenderCompatible(teamA: TeamProfile, teamB: TeamProfile): boolean {
  const totalMale = teamA.maleCount + teamB.maleCount;
  const totalFemale = teamA.femaleCount + teamB.femaleCount;
  return totalMale === totalFemale;
}
```
